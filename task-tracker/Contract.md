# Module 3 Kanban Board — Behavior Contract

Locks in the current behavior of the Kanban board before refactoring. Board logic
lives in the inline `<script>` in `frontend/index.html` (the sibling `frontend/app.js`
is not loaded). The backend serves from `http://localhost:8000`.

| ID | Behavior | How to check manually | Pass/Fail notes |
|----|----------|-----------------------|-----------------|
| **B1** | Three status columns render with correct counts | Start the backend, open the page. Confirm exactly three columns — **To Do**, **In Progress**, **Done** — appear left to right. For each, count the cards and compare to the number in the pill (`[data-count]`) in the column header. Cross-check against `GET /tasks` grouped by `status`. | Only `ToDo`/`InProgress`/`Done` are shown; tasks with any other status are silently dropped (won't appear and won't be counted). |
| **B2** | Cards sort by priority inside each column | In any column with ≥2 cards, read top to bottom: **High → Medium → Low** (left border + pill color). For same-priority cards, they should order by ascending `id`. | Sort is High=0, Med=1, Low=2, tie-broken by `id`. Any priority value outside these three would land as `undefined` in ranking — verify none exist. |
| **B3** | Loading state appears before tasks load | Throttle network (DevTools → Network → Slow 3G) or add latency, then reload. Before cards appear you should see a centered panel with a spinner and "Loading tasks…". The board columns should be hidden during this. | Loading panel has `role="status"`. It's brief on a fast local backend — throttling makes it observable. |
| **B4** | Empty columns remain visible | Ensure at least one status has zero tasks (e.g. no Done tasks). Confirm that column still renders with its header, a count of **0**, and a dashed **"No tasks"** placeholder — it must not collapse or disappear. | Placeholder (`.card-list-empty`) also preserves the drop zone for drag/drop. Check all three can be empty independently. |
| **B5** | Error state appears when the backend is stopped | Stop the backend, then reload the page. You should see a panel with border in the error color, the message "Couldn't load tasks…", and a **Retry** button. Click **Retry** while backend is still down → error persists. Restart backend, click **Retry** → board loads. | Error panel has `role="alert"`. Triggered by fetch throw or any non-`ok` response from `GET /tasks`. |
| **B6** | Valid drag sends PATCH and updates the board | With backend running and DevTools Network open, drag a card from one column to another. Card should move immediately (optimistic). Confirm a `PATCH /tasks/{id}` fires with body `{"status":"<target>"}` and returns 200. Reload → card stays in the new column. Counts on both columns update. | Dropping a card back on its own column is a no-op (no request). Optimistic move happens before the server responds. |
| **B7** | Invalid drag / server 422 reverts and shows the server message | Force a rejection: either stop the backend mid-session then drag (network failure path), or have the backend reject the status with a 422/4xx. The card should snap **back** to its original column and a red toast should appear with the server's message (from `detail`), or "Couldn't reach the server…" on network failure. | Revert re-renders from the reverted `tasks` array. Toast auto-dismisses after 5s. Verify counts also revert. |
| **B8** | New Task and Edit modal flows still work, incl. title validation and dismissal | **Create:** click **New Task** → modal opens titled "New Task", Title focused. Submit empty title → inline "Title is required", no request sent. Fill a valid title, Save → `POST /tasks` 200, modal closes, board refreshes with the new card. **Edit:** click a card's **Edit** → modal titled "Edit Task" pre-filled with that task's fields; change a field, Save → `PATCH /tasks/{id}`, board updates. **Dismissal:** each of Cancel, ✕, overlay click, and **Escape** closes the modal without saving. **Server 422:** submit data the backend rejects → field-level errors show and modal stays open. | Client blocks empty/whitespace-only title before any request. Blank assignee is sent as `null`. 422 errors route to matching field slots via `loc`; unmapped errors fall to the form banner. |
